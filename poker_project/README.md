# Poker AI Project

This project provides a complete pipeline for creating, training, and evaluating a Texas Hold'em poker AI. It uses `PyPokerEngine` for game simulation and `scikit-learn` for model training.

## Project Structure

The project is organized into several key files:

-   `generate_data.py`: This script runs a poker simulation with simple, rule-based bots to generate a dataset of game states and actions. The data is saved to `poker_data.csv`.
-   `train.py`: This script reads the data from `poker_data.csv`, preprocesses it into a feature set, and trains a `RandomForestClassifier` model. The trained model and a label encoder are saved to `poker_model.joblib` and `label_encoder.joblib` respectively.
-   `evaluate_model.py`: This script loads the trained model and evaluates its performance by having it play against the simple bots in a new simulation.
-   `poker_data.csv`: The dataset generated by `generate_data.py`.
-   `poker_model.joblib`: The trained machine learning model saved by `train.py`.
-   `label_encoder.joblib`: The label encoder for the model's actions, saved by `train.py`.

## Setup

To run this project, you need to install the required Python libraries.

```bash
pip install PyPokerEngine pandas scikit-learn
```

## Usage

The project is designed to be run in a specific sequence. Follow these steps to generate data, train a model, and evaluate it.

### 1. Generate Data

First, run the data generation script. This will simulate a game for 1000 rounds and create the `poker_data.csv` file.

```bash
python poker_project/generate_data.py
```

### 2. Train the Model

Next, run the training script. This will read the generated data, train the model, and save `poker_model.joblib` and `label_encoder.joblib`.

```bash
python poker_project/train.py
```

### 3. Evaluate the Model

Finally, run the evaluation script to see how your trained model performs against the other bots.

```bash
python poker_project/evaluate_model.py
```
